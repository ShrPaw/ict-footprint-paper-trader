## What Was Done

### 1. Per-Asset Signal Thresholds
Added `minConfluenceScore` and `minSoloScore` to each asset's `daytrade` config in `assetProfiles.js`:
- BTC: 0.55 / 0.70 (low trade volume, high WR — slightly tighter)
- ETH: 0.60 / 0.75 (moderate, baseline)
- SOL: 0.52 / 0.68 (extreme vol — looser, compensated by tight SL)
- XRP: 0.62 / 0.78 (speculative noise — tightest)

Both `DaytradeMode._scoreSignal()` and `backtest.js._evaluateSignal()` now read per-asset 
thresholds with fallback to global config defaults.

### 2. Fixed Precomputed Confidence Values
The Precompute.js feature extractors were producing raw confidence values (0.04-0.55) that got 
multiplied by source weights (0.4 ICT, 0.6 FP), making scores impossible to reach thresholds.

Fixes applied to Precompute.js:
- **Order Blocks**: `confidence = Math.min(0.5 + (move - 0.003) / 0.017 * 0.5, 1.0)`
  - Maps 0.3%-2% moves to 0.50-1.0 range
  - Old: `Math.min(move * 100, 1.0)` → 0.3% move = 0.30 confidence
- **Liquidity Sweeps**: `confidence = Math.min(0.65 + (wickRatio - 0.6) / 0.4 * 0.35, 1.0)`
  - Maps wick ratio 0.6-1.0 to 0.65-1.0 range
  - Old: `Math.min(wickRatio, 1.0)` → 0.6 ratio = 0.60 confidence
- **OTE zones**: bumped from 0.7 to 0.8

### 3. Fixed Backtest Scoring
In `_evaluateSignal()`, normalized combinedScore by dividing by source weight:
- ICT: `normalized = (sig.confidence / ictWeight)` then apply weight
- FP: `score = divSignal.confidence / fpWeight` then apply weight

This ensures precomputed scores match the live DaytradeMode scale.

1. **Fees dominate** — $37K fees vs ~$3K net loss. System trades for exchanges.
2. **Overtrading** — 5225 trades / 4 years = ~108/month. Original SPOT was ~40/month.
3. **Trailing stops = the edge** — 100% WR on trailing_sl across all assets.
4. **Regular stops = noise traps** — 7-14% WR on SL hits. Stops too tight for futures volatility.
5. **RANGING is toxic on futures** — ETH: -$1,869 in RANGING (45% WR). Should block it.
