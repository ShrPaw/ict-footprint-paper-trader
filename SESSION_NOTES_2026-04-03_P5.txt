   - Answers: "Is this move already over?"
   - 6 detection checks:
     * Volatility spike (ATR z-score)
     * Price extension from mean
     * Stacked delta exhaustion
     * Recent move magnitude
     * Volume climax
     * Momentum deceleration

## ARCHITECTURAL CHANGES

### Before (v5.0)
```
Model → Signal → Execute
                    ↓
              Emergency Stop (0% WR)
```

### After (v6.0)
```
Model → Signal → Exhaustion Gate → Portfolio Risk → Execute
              ↓ (blocked)        ↓ (blocked)
              NO TRADE           NO TRADE
              ↓ (adjusted)
              Reduced Size
```

### Signal Flow (New)
```
1. Model generates signal (per-asset alpha engine)
2. ExhaustionDetector.check() — is this move already over?
   → Critical triggers: ATR-z, price extension, recent move
   → Secondary triggers: stacked delta, volume climax, momentum decay
   → BLOCK if any critical OR 2+ secondary
3. PortfolioRiskManager.validate() — is the portfolio safe?
   → Portfolio DD check
   → Daily loss check
   → Emergency stop clustering check
   → Consecutive loss check
   → Per-asset emergency limit check
   → Adjustments: confidence decay, DD proximity, correlation
4. Execute with adjusted position size
5. PortfolioRiskManager.recordTrade() — track for future decisions
```
