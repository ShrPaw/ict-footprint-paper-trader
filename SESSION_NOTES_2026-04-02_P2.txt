## Who

## What Was Done

### 1. Fixed CRITICAL Bug in backtest.js `_buildContext()`
- Variable `regime` was used but NEVER DEFINED in the function scope
- It referenced `regimeData.regime` at the return but the local `regime` variable was missing
- This would cause `ReferenceError` or silently use wrong regime filtering
- **Fix:** Added `const regime = regimeData.regime;` after `regimeData` check

### 2. Regime-Dependent Slippage
- Old: flat 0.005% slippage for ALL regimes
- Problem: VOL_EXPANSION has wider spreads (fast moves), RANGING is tight
- **Fix:** Added `slippageByRegime` to config.js:
  - RANGING: 0.003% (tight, liquid)
  - VOL_EXPANSION: 0.015% (wide, fast moves)
  - TRENDING_UP/DOWN: 0.008%
  - LOW_VOL: 0.003%
  - ABSORPTION: 0.010%
- Applied to: PaperEngine.js (open, close, partial TP), backtest.js (open, close, partial TP)

### 3. Daily Trade Limit — NOW ENFORCED
- Config already had `maxDailyTrades: 2` but it was NEVER CHECKED
- PaperEngine.js tracked `dailyTradeCount` but never incremented or enforced
- **Fix:**
  - PaperEngine: added check `if (this.dailyTradeCount >= config.engine.maxDailyTrades)` + increment on open
  - backtest.js: added `dailyTradeCount` tracking, reset on new day, check before open, increment on open

### 4. Session Hard-Gating (BTC + XRP)
- Old: session weights were SOFT multipliers (0.7-1.3) — bad sessions still traded if signal was strong
- **Fix:** Added `allowedSessions` array to asset profiles:
  - BTC: `['ny', 'overlap']` — only institutional hours
  - XRP: `['overlap', 'ny']` — highest liquidity sessions
  - ETH: no gate (all sessions work)
  - SOL: no gate (works everywhere)
- Enforced in: backtest.js `_buildContext()` and DaytradeMode.js `generateSignal()`
