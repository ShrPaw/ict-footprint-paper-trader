## What Was Built This Session

### 1. Pine Script Strategy (`strategy/ICT_Footprint_Strategy.pine`)
Full TradingView strategy replicating the backtested logic:
- Regime detection: ATR percentile, ADX, Bollinger squeeze, volume ratio
- ICT signals: Order Blocks (swing high/low zones), OTE retracement zones (0.618-0.786), Liquidity sweeps (wick ratio > 0.6)
- Footprint proxy: volume + candle structure (real footprint needs trade-level data, not available in Pine)
- Per-asset regime blocking via input settings:
  - BTC: blocks RANGING (loses -$728), works in VOL_EXPANSION
  - ETH: no blocks (works in RANGING + VOL_EXP)
  - SOL: no blocks (works in RANGING)
  - XRP: blocks RANGING (loses -$282), works in VOL_EXPANSION
  - TRENDING_DOWN and LOW_VOL blocked globally for all assets
- Killzones: London (06-12), NY (12-18), Asia (22-04) UTC
- Weekend filter (skip weekends)
- EMA alignment (EMA9, EMA21, EMA50)
- Confluence scoring with configurable weights
- Risk: 0.5% per trade, SL 0.5x ATR, TP 2.0x ATR, trailing stop
- Info table overlay showing regime, ADX, ATR%, scores, killzone status
- Alerts fire on entry with JSON payload for webhook

### 2. Webhook Server (`webhook-server/server.js`)
Node.js HTTP server that receives TradingView alerts:
- Listens on port 3450
- POST /webhook — receives entry/close signals from TradingView
- Paper trade engine: opens positions, tracks PnL, fees (0.05% taker)
- Position sizing: 0.5% risk per trade
- Cooldown: 45 minutes between signals per symbol
- API endpoints:
  - GET /api/stats — balance, PnL, win rate, profit factor, drawdown
  - GET /api/positions — open positions
  - GET /api/trades — last 50 trades
  - POST /api/reset — reset to $10,000
- Persists trades to paper_trades.json

### 3. Dashboard (`webhook-server/dashboard.html`)
Dark-themed web dashboard:
- Auto-refreshes every 3 seconds
- Stats cards: balance, PnL, win rate, profit factor, max drawdown, fees
- Open positions with entry/SL/TP/regime
- Trade history table with PnL colors
- Reset button

## Architecture (TradingView Paper Trading Flow)

```
TradingView Chart
  └─ Pine Script strategy runs on chart
     └─ Detects regime, ICT signals, confluence
     └─ Executes paper trades on TradingView
     └─ Fires alert with JSON payload
        └─ Webhook POST → http://server:3450/webhook
           └─ Server tracks positions, PnL, fees
           └─ Dashboard at http://server:3450/
```

Two independent paper trading systems:
1. TradingView's own paper trading (visible on chart)
2. Local server tracker (separate PnL tracking with dashboard)

## How To Run
```bash
cd tradingview-paper-trade
npm install
npm run server          # starts webhook server + dashboard at :3450
```

Then on TradingView:
1. Open Pine Editor → paste ICT_Footprint_Strategy.pine → Add to chart
2. Select Asset Profile in strategy settings (BTC/ETH/SOL/XRP)
3. Create alert → webhook URL: http://YOUR_IP:3450/webhook
4. Use {{strategy.order.alert_message}} as alert message
