## Architecture: Before vs After

### BEFORE (Single Strategy)
```
DaytradeMode._scoreSignal()
  → Same logic for all 4 assets
  → Only thresholds/weights differ
  → Order blocks treated the same on SOL and BTC
  → Result: emergency stops, curve-fitting via parameters
```

### AFTER (Multi-Model)
```
ModelRouter
  → SOLModel: Retail momentum (stacked delta, volatility filter, late-move guard)
  → BTCModel: Institutional structure (OB, sweep, OTE, EMA stack)
  → ETHModel: Hybrid confluence (mandatory ICT + flow agreement)
  → XRPModel: Extreme events only (all 3 conditions required)
```

## Per-Asset Model Design

### SOL — Retail Momentum Engine
- **Edge source:** Momentum + order flow imbalance
- **Core features:** Stacked delta imbalance (3+), short-term momentum, volatility expansion filter
- **Filters:** ATR z-score > 2 = no trade, price > 2 ATR from mean = no trade
- **Avoids:** Order blocks (retail doesn't respect them), OTE (too slow for SOL speed)
- **Regime:** RANGING is gold (+$5,832 historically), TRENDING_UP continuation works

### BTC — Institutional Structure Engine
- **Edge source:** ICT structure + controlled order flow
- **Core features:** Order block retest, OTE zone, liquidity sweep reversal, full EMA stack
- **Filters:** ATR z-score > 1.5 = avoid (volatility spike = knife catching)
- **Requires:** Full EMA 9>21>50 alignment (not just price > EMA21)
- **Regime:** TRENDING_UP only (blocked in RANGING historically)

### ETH — Hybrid Confluence Engine
- **Edge source:** Confluence ONLY — no solo signals
- **Core features:** ICT zone (OB/sweep/OTE) + flow signal (stacked delta/divergence)
- **Rule:** No ICT zone = no trade. No flow signal = no trade. Both must agree on direction.
- **Regime:** TRENDING_UP primary, VOL_EXP gated

### XRP — Extreme Event Engine
- **Edge source:** Extreme divergence ONLY
- **Core features:** Delta z-score > 2.0, volume spike > 2x, price stall (body < 50% avg)
- **Rule:** ALL THREE conditions must be met. Missing any = no trade.
- **Regime:** VOL_EXPANSION only (blocked everywhere else)
- **Direction:** Reversal only (delta direction tells what's being absorbed)

## Shared Utilities (BaseModel)

Each model inherits:
- `checkKillzone()` — session filtering
- `isWeekend()` — weekend filter
- `checkSessionGate()` — per-asset allowed sessions
- `checkRegimeBlock()` — per-asset blocked regimes
- `computeATRzScore()` — volatility expansion detection
- `priceDistanceFromMean()` — late-move filter
- `buildSignal()` — standardized signal construction with per-asset SL/TP
