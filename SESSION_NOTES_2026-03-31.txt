## What Was Fixed (Two Rounds)

### Round 1 (Session #4): `.filter()` → incremental index tracking
- Was: `candles5m.filter(c => c.timestamp <= timestamp)` on 446K items × 148K iterations
- Changed to: `while` loops advancing `hIdx`/`m5Idx` indices

### Round 2 (Session #5): `slice()` → cached windows on 1h boundaries only
- Was: `candles1h.slice(0, hIdx + 1)` on every 15m candle (148K times)
- Now: only slice when `hIdx` changes (new 1h candle closes, ~37K times)
- When in a position: NO slicing at all — exit check just needs current candle OHLC
- Added progress indicator every 10K candles

## Why It's Still Slow
The fundamental issue: even with these optimizations, the backtest has to:
1. Iterate 148K 15m candles
2. On each: advance indices, check exits, track equity
3. On ~37K of those: slice 1h window (up to 37K items) for regime + signal detection
4. The regime detector, ICT analyzer, and footprint analyzer each run full computations on the sliced window

The analyzers recompute from scratch every time:
- `RegimeDetector.detect()` — recalculates EMA, ADX, ATR, Bollinger on full window
- `ICTAnalyzer.analyze()` — scans for FVGs, OBs, liquidity sweeps
- `RealFootprintAnalyzer.analyze()` — rebuilds delta, volume profile

These are O(n) per call on the window size, and they're called ~37K times.

## What Would Actually Fix It
**Option A: Pass indices, not arrays**
- Change analyzer signatures to accept `(allCandles, startIdx, endIdx)` instead of `(window)`
- Maintain incremental state (EMA values, ADX state) and update incrementally
- Major refactor of RegimeDetector, ICTAnalyzer, RealFootprintAnalyzer

**Option B: Reduce signal check frequency**
- Only check for signals when a new 1h candle closes (every 4th 15m candle)
- Reduces analyzer calls from ~37K to ~9K
- Simpler, no signature changes

**Option C: Pre-compute indicators**
- Pre-compute EMA9/21/50, ATR, ADX arrays for 1h candles upfront
- Look up values by index instead of recalculating
- Medium effort, big speedup
