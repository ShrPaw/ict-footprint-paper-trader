## Root Cause: O(n²) Window Building
In `engine/backtest.js`, the main loop does this on EVERY 15m candle:
```js
const window5m = candles5m.filter(c => c.timestamp <= timestamp);  // 446K items
const window1h = candles1h.filter(c => c.timestamp <= timestamp);  // 37K items
```
This is O(n) per iteration × 148K iterations = impractical.

## Fix Applied (needs testing)
Changed to incremental index tracking:
```js
let hIdx = 0;
let m5Idx = 0;
for (let i = 50; i < candles15m.length; i++) {
  while (hIdx < candles1h.length - 1 && candles1h[hIdx + 1].timestamp <= timestamp) hIdx++;
  while (m5Idx < candles5m.length - 1 && candles5m[m5Idx + 1].timestamp <= timestamp) m5Idx++;
  const window1h = candles1h.slice(0, hIdx + 1);
  const window5m = candles5m.slice(0, m5Idx + 1);
  // ... rest unchanged
}
```
Also pre-computes regime distribution from 1H candles separately (was being tracked inside the 15m loop via redundant regime.detect calls).

**Logic is identical** — every 15m candle still checked for exits, signals still generated at same points. Just removed the wasteful repeated filtering.

## Statistical Rigor Requirements (from Nicolas)

### Non-Negotiable
1. **Out-of-sample testing** — Train on 2022-2024, test on 2025+
2. **Walk-forward, not curve-fit** — Already walk-forward candle-by-candle, regime blocking discovered from results (honest)
3. **Include fees + slippage + funding rates** — Fees/slippage done, funding rate implemented (`--funding 0.0001`) but NOT yet tested on futures data
4. **Enough trades** — Need 100+ trades per regime per asset. Going to 2022 on futures should give this
5. **Survive different market regimes** — Must work in:
   - 2022 bear crash (Luna, FTX)
   - 2023 slow grind up
   - 2024 bull run (BTC ETF euphoria)
   - 2025-26 choppy

### What Separates Good from Great
6. **Stability across assets** — XRP PF 2.27 vs BTC PF 1.53 is concerning. If one asset carries the system, red flag
7. **Max DD vs return** — Rule: max DD < 50% of total return
8. **No single trade dependency** — Check largest win. Code already computes `pnlWithoutLargest`
9. **Consistent WR across time periods** — Monthly breakdowns matter (already implemented)
10. **Expectancy > $0 after realistic costs** — ~$18/trade is thin. Funding rates could flip this

### Required Output Per Backtest
- Year-by-year PnL breakdown (2022 bear, 2023 grind, 2024 bull, 2025-26 chop)
- Regime distribution (% of time each regime active)
- Monthly return distribution (median, best/worst, profitable months %)
- Consecutive loss streaks
- With and without funding rates
- Single trade dependency check
- DD/Return ratio

**All of this is already coded in backtest.js** — just needs to actually run.
