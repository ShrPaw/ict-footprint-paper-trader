**Result: 50 → 14 trades (72% drop) — VIOLATES >70% hard constraint**

The saturated filter alone shouldn't cause this (only blocks stacked >= 5).

But the total signal count dropped from 26 to 8 — more than just the saturated filter explains.

**Partial diagnosis:** Debug output showed SOL DEBUG lines for SATURATED blocks, but the 
total signal count drop suggests something else is also filtering. Possibly:
- The context classification is being called and the `flowCtx` variable is somehow 
  affecting control flow
- Or the model is returning null earlier (in gates/features) for other reasons
- Need deeper debugging
